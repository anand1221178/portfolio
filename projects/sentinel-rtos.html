Project Overview

A fully preemptive real-time operating system built from scratch on bare metal for the STM32F401RE (ARM Cortex-M4). No HAL, no FreeRTOS, no libraries — every peripheral driver, scheduler, and synchronization primitive is implemented from first principles against the STM32 reference manual and ARM Architecture Reference Manual.

The RTOS includes a priority-based preemptive scheduler with priority inheritance to prevent priority inversion, PendSV-driven context switching in ARM assembly, and a full suite of synchronization primitives (mutexes, semaphores, message queues). MicroBench, a custom benchmarking suite, measures context switch latency, interrupt response time, and mutex overhead, demonstrating competitive performance against FreeRTOS on identical hardware.

Key Features

Preemptive Scheduler

Priority-based preemptive scheduler with priority inheritance protocol. PendSV-driven context switching in ARM assembly preserves full register state.

Bare-Metal Drivers

UART, SPI, I2C, PWM, GPIO, and ADC drivers written via direct register manipulation against STM32F401RE datasheets. Zero HAL dependency.

Sync Primitives

Mutexes with priority inheritance, counting semaphores, and message queues. All implemented from scratch with proper interrupt-safe critical sections.

Sensor Integration

HC-SR04 ultrasonic and PIR motion sensors with PID-controlled servo sweep. ESP8266 WiFi telemetry with custom binary protocol for real-time radar visualization.

Technical Implementation

1. PendSV Context Switching

Context switching is triggered by the PendSV exception, the lowest-priority interrupt in ARM Cortex-M. This ensures context switches only occur when no other ISRs are active. The handler saves the current task's registers to its stack and restores the next task's registers.

PendSV_Handler:
    CPSID   I                   @ Disable interrupts
    MRS     R0, PSP             @ Get current PSP
    STMDB   R0!, {R4-R11}      @ Save R4-R11 to task stack

    BL      scheduler_next_task @ Get next task (C function)

    LDMIA   R0!, {R4-R11}      @ Restore R4-R11 from new stack
    MSR     PSP, R0             @ Set new PSP
    CPSIE   I                   @ Re-enable interrupts
    BX      LR                  @ Return (hardware restores rest)

2. Priority Inheritance

When a high-priority task blocks on a mutex held by a lower-priority task, the holder's priority is temporarily boosted to prevent priority inversion. The original priority is restored when the mutex is released.

void mutex_lock(mutex_t *m) {
    disable_interrupts();
    if (m->locked && m->owner != current_task) {
        // Boost owner priority to prevent inversion
        if (current_task->priority > m->owner->priority)
            m->owner->priority = current_task->priority;
        // Block current task on this mutex
        task_block(current_task, &m->wait_queue);
        schedule();
    }
    m->locked = 1;
    m->owner = current_task;
    enable_interrupts();
}

3. MicroBench: Performance Validation

MicroBench is a custom benchmarking suite that measures key RTOS performance metrics using the ARM DWT cycle counter for nanosecond-precision timing. Results are compared against FreeRTOS running on identical hardware.

Results

Context Switch Performance

MicroBench measures context switch latency, interrupt response time, and mutex acquisition overhead using the ARM DWT cycle counter. Sentinel RTOS demonstrates competitive performance against FreeRTOS on identical STM32F401RE hardware.

Real-World Sensor Fusion

Integrated HC-SR04 ultrasonic and PIR motion sensors with a PID-controlled servo sweep loop. ESP8266 WiFi module sends telemetry using a custom binary protocol to a Python ground station that renders a real-time radar display.

Conclusion

Sentinel RTOS demonstrates mastery of embedded systems fundamentals — ARM architecture, interrupt handling, real-time scheduling, and peripheral interfacing — all implemented without any abstraction layers. The project serves as a complete reference implementation of a preemptive RTOS suitable for resource-constrained embedded systems.
