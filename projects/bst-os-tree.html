Project Overview

This project provides a from-scratch implementation and analysis of Binary Search Trees (BST) and Order-Statistic Trees (OS-Tree) in C, following the specifications in CLRS. The project includes all core BST operations and the augmented OS-SELECT and OS-RANK operations, which allow for finding the i-th smallest element and determining an element's rank in O(log n) time.

Through comprehensive empirical testing, the project verifies the theoretical complexities and quantifies the performance overhead of augmenting a data structure. The results show that OS-Trees incur a minimal 8-10% performance overhead on insertion and deletion operations while providing powerful rank-based query capabilities.

Key Features

BST Implementation

A from-scratch implementation of all core BST operations as specified in CLRS.

Size Augmentation

Each node in the OS-Tree is augmented to store its subtree size, enabling rank queries.

O(log n) OS-SELECT

An efficient implementation of the OS-SELECT operation to find the i-th smallest element.

O(log n) OS-RANK

An efficient implementation of the OS-RANK operation to determine the rank of an element.

2. Order-Statistic Tree (OS-Tree)

The BST is augmented to create an OS-Tree. Each node is modified to include a `size` field, which stores the number of nodes in its subtree. This `size` field is maintained during insertion and deletion operations.

Conclusion

This project successfully implements and analyzes BSTs and OS-Trees. The empirical results confirm the theoretical complexities and demonstrate that the 8-10% performance overhead for augmenting the data structure is a small price to pay for the powerful O(log n) rank-based queries that OS-Trees provide. This serves as a practical example of the trade-offs involved in data structure design.
