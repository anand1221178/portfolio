Project Overview

This project implements and analyzes three matrix multiplication algorithms from CLRS in C: the naive O(n³) algorithm, a recursive O(n³) divide-and-conquer approach, and Strassen's O(n^2.81) algorithm. Through empirical testing up to n=4096, the project demonstrates that Strassen's algorithm, despite its superior asymptotic complexity, is 1.59x slower than the naive implementation due to memory overhead and cache inefficiency.

The key finding is that the theoretical advantage of Strassen's algorithm is overshadowed by large constant factors and poor memory hierarchy utilization in practice. The estimated crossover point where Strassen's algorithm would outperform the naive implementation is around n≈16,000, which is often an impractically large matrix size. This project highlights the critical importance of empirical analysis in understanding real-world algorithm performance.

Key Features

Naive O(n³)

The standard triple-nested loop implementation, serving as a cache-friendly baseline.

Recursive O(n³)

A divide-and-conquer approach that makes 8 recursive calls, demonstrating the overhead of recursion.

Strassen's O(n^2.81)

An optimized divide-and-conquer algorithm that uses 7 recursive calls instead of 8.

Empirical Analysis

Detailed benchmarking of all three algorithms for matrix sizes up to n=4096.

Technical Implementation

1. Naive Matrix Multiplication (O(n³))

The baseline is the standard triple-nested loop algorithm. Its sequential memory access pattern makes it highly cache-efficient. 

// From src/naive.c
void naive_multiply(int n, double **A, double **B, double **C) {
    for (int i = 0; i < n; i++) {
        for (int j = 0; j < n; j++) {
            C[i][j] = 0;
            for (int k = 0; k < n; k++) {
                C[i][j] += A[i][k] * B[k][j];
            }
        }
    }
}

2. Recursive Divide-and-Conquer (O(n³))

This algorithm divides the matrices into quadrants and performs 8 recursive multiplications. It has the same asymptotic complexity as the naive algorithm but with significantly more overhead. 

// From src/recursive.c
void recursive_multiply(int n, double **A, double **B, double **C) {
    if (n == 1) {
        C[0][0] = A[0][0] * B[0][0];
        return;
    }
    // ... 8 recursive calls ...
}

3. Strassen's Algorithm (O(n^2.81))

Strassen's algorithm reduces the number of recursive calls from 8 to 7, which leads to a better asymptotic complexity. However, it requires additional matrix additions and subtractions, and significant temporary matrix allocations. 

// From src/strassen.c
void strassen_multiply(int n, double **A, double **B, double **C) {
    if (n <= 128) {  // Base case: switch to naive
        naive_multiply(n, A, B, C);
        return;
    }
    // ... 7 recursive calls and combination logic ...
}

Analysis and Insights

Memory Overhead

Issue: Strassen's algorithm allocates numerous temporary matrices at each recursive step, leading to significant memory overhead and cache thrashing.

Impact: At n=4096, Strassen's algorithm uses approximately 1.5GB of temporary storage, compared to 128MB for the naive algorithm. This destroys cache locality and results in a 3x slowdown.

Crossover Point

Analysis: By extrapolating the growth rates, the crossover point where Strassen's algorithm would become faster than the naive implementation is estimated to be at n≈16,000. For most practical applications, this is an impractically large matrix size.

Conclusion

This project highlights the crucial difference between theoretical and practical algorithm performance. While Strassen's algorithm is asymptotically faster, the naive algorithm is significantly faster in practice for all but the largest matrix sizes. This is due to the naive algorithm's superior cache locality and lower memory overhead. The project serves as a clear demonstration that Big-O notation does not tell the whole story, and empirical testing is essential for understanding real-world performance.
