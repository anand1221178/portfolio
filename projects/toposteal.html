Project Overview

TopoSteal is a topology-aware work-stealing scheduler that discovers the CPU cache and NUMA hierarchy at runtime using hwloc and biases victim selection toward topologically close neighbours using inverse-square distance weighting (1/d²). A background thread polls hardware performance counters via Linux's perf_event_open syscall and dynamically adjusts steal probabilities based on observed cache miss rates.

Evaluated on a dual-socket Intel Xeon E5-2690v3 (24 cores, NUMA distances 10/21) from the CHPC Lengau cluster. Achieved 1.15–1.18x speedup over uniform stealing while increasing same-socket steal locality from 50% to 85%. Outperformed OpenMP schedule(static) with OMP_PROC_BIND=close by 5% on SpMV with audikw_1 (943K rows, 77M nonzeros) from SuiteSparse.

Key Features

Topology Discovery

Discovers CPU cache hierarchy at runtime via hwloc and builds a pairwise distance matrix between all cores.

PMU Feedback Loop

Background sampler reads hardware cache miss counters every 10ms via perf_event_open and dynamically rebuilds steal probability tables.

Lock-Free Deques

Chase-Lev work-stealing deques with C11 atomics and sequential consistency fences. Stress-tested with 81 million tasks, zero lost.

NUMA-Aware Stealing

Inverse-square distance weighting biases victim selection 85% toward same-socket cores, reducing cross-socket DRAM traffic.

Technical Implementation

1. Weighted Victim Selection

Given the distance matrix D from hwloc, steal weights are computed using inverse-square weighting: wij = 1/dij². Weights are normalized per-row to form a CDF. Victim selection performs a single pass over the CDF with a uniform random sample.

int weights_pick_victim(weights_t *w, int worker_id, unsigned int *seed) {
    float r = (float)rand_r(seed) / RAND_MAX;
    for (int j = 0; j < w->num_workers; j++) {
        if (j == worker_id) continue;
        if (r < w->steal_thresholds[worker_id][j])
            return j;
    }
    return -1;
}

2. PMU-Driven Dynamic Feedback

Every 50ms, the feedback system reads per-worker cache miss rates and adjusts the effective distance: d'ij = dij × (1 + normalized_miss_ratej). Workers under high cache pressure become less attractive steal targets.

Results

Pointer Chase Microbenchmark

Across 5 configurations and 50 trials, TopoSteal achieved 1.15–1.18x speedup over uniform stealing. Same-socket steal locality increased from 50% to 85%, matching the theoretical prediction from 1/d² weighting.

SpMV on SuiteSparse (audikw_1)

On real-world SpMV with a 77M-nonzero matrix from an Audi car body simulation, TopoSteal achieved 7.46 GFLOP/s — outperforming OpenMP schedule(static) with OMP_PROC_BIND=close by 5%. Topology-aware work-stealing surpasses production-quality static scheduling when task placement creates natural imbalance.

Tail Latency Analysis

Cross-socket steals incur a 28–31% per-task latency penalty. TopoSteal reduces cross-socket steal exposure from 50% to 14%, cutting the number of tasks hitting the expensive remote DRAM path by 71%.

Conclusion

TopoSteal demonstrates that topology-aware victim selection yields measurable improvements on commodity NUMA hardware. The project includes a full IEEE-format technical paper and is being extended for multi-socket evaluation in collaboration with CHPC and Georgia Tech researchers, targeting submission to SC (Supercomputing Conference).
